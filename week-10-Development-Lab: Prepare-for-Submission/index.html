Week 10 — Development Lab: Prepare for Submission

This final session is a dedicated support and troubleshooting space. No new coding is introduced. Instead, it’s your chance to review your code, prepare your submission, and resolve any last-minute issues.

Submission Instructions

  1. Submit a single ZIP file (less than 100MB) via Turnitin.
  2. The ZIP should contain your HTML and CSS files for your Final Project.
  3. Name your ZIP file: XXXXXX_MA1800_FinalProject.zip, replacing XXXXXX with your candidate number.
  4. Ensure your code includes code comments.
  5. Include a README.md that contains:
    • A link to your project on GitHub Pages.
    • A description of your project idea, context, functionality, and challenges encountered.
    • A summary of seminar case studies that influenced your work.
    • A reflection on how you would extend the project with more time or resources.
    • A list of assets you created and those sourced online.

What to Do in This Session

  • Review your code by testing different functionality in multiple browsers.
  • Check that all key aspects of your web artwork (navigation, layout, styling, interactivity) work properly.
  • Confirm your GitHub Pages link loads correctly.
  • Ask for help with unresolved bugs, errors, or file organisation.
  • Exchange feedback with peers if time allows.

Reminder: Use this final session to ensure your project is complete, functional, and submission-ready.
